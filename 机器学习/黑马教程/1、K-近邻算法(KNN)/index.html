1、K-近邻算法（K-Nearest Neighbor，KNN）

定义：在一个样本空间中，K个最相似（距离样本最近）的样本中的大多数属于某一个类别，则该样本也属于这个类别

举例：

通过算法求出9号电影的类别

算法流程：

  1. 计算已知类别数据集中的点与当前点之间的距离
  2. 按距离递增次序排序
  3. 选取与当前点距离最近的k个点
  4. 统计前k个点所在类别出现的频率
  5. 返回k个点出现评率最高的类别作为当前点的预测分类

这里以上图为例，我用欧氏距离计算其各个的距离，为['21.47', '52.01', '43.42', '40.57', '34.44', '43.87', '18.55', '23.43']

其中最近的5个点（K=5时）为7,1,8,5,4

故将其分类到喜剧片中

以下为欧氏距离公式

二维平面上a(x1,y1)与b(x2,y2)间的欧氏距离

$ {d}_{12} = \sqrt{{\left( {x}_{1} - {x}_{2}\right) }^{2} + {\left( {y}_{1} - {y}_{2}\right) }^{2}} $

n维空间点a(X11,x12,…,x1n)与b(X21,x22,…,X2n)间的欧氏距离（两个n维向量)：

$ {d}_{12} = \sqrt{\mathop{\sum }\limits_{{k = 1}}^{n}{\left( {x}_{1k} - {x}_{2k}\right) }^{2}} $

马氏距离是基于样本分布的一种距离。

马氏距离定义：设总体G为m维总体（考察m个指标），均值向量为μ=（μ1，μ2，… …，μm，）,协⽅差阵为 ∑=（σij），则样本X=（x1，x2,… …，Xm，）与总体G的⻢⽒距离定义为：

$ {D}_{M}\left( \mathbf{x}\right) = \sqrt{{\left( \mathbf{x} - \mu \right) }^{T}{\mathbf{\sum }}^{-1}\left( {\mathbf{x} - \mu }\right) } $

其中：

  • x为数据点向量
  • μ是数据集的均值向量
  • ∑是数据集的协方差矩阵
  • Σ−1 是协方差矩阵的逆矩阵

4、K值的选择

  1. 选择较小的K值，相当于用较小范围的实例来进行预测：
  • “学习”的近似误差会减小，但估计误差会增大
  • 换句话说就是K值的减小意味着整体模型会变复杂，容易发生过拟合
  1. 选择较大的K值，相当于用较大范围的实例来进行预测：
  • 优点是减少学习的估计误差，但会增大近似误差，这时与输入实例较远（不相似）的训练实例会起作用，使预测发生错误，容易欠拟合
  • 且K值的增大会使模型变得简单
  1. 若K=N（样本个数）
  • 此时无论输入实例是什么，都只是简单的预测它属于训练实例中最多的类（已经无视距离远近了）。
  • 近似误差：
    • 对现有训练集的训练误差，关注训练集
    • 如果近似误差过小则可能会出现过拟合的现象
  • 估计误差：
    • 对测试集的测试误差，关注测试集
    • 估计误差小说明对未知数据的预测能力好，本身接近最佳模型

5、KD树

在实现KNN算法时，主要考虑的是如何对训练数据进行快速的K近邻搜索

KNN最简单的实现是线性扫描（穷举搜索），即计算输入实例与每一个训练实例的距离，计算并存储，然后查找K近邻。时间复杂度O(DN²)

5.1 简介

为了每次都要重新计算一遍距离，算法会把距离信息存放在一棵树中，这样在计算之前查询距离信息，避免重复计算。基本原理是：如果A与B距离较远，而B与C的距离较近，则得出A与C的距离也较远，则可以跳过这些距离远的点。优化后的时间负责度为O(DNlog(N))。

2、最近领域搜索（Nearest-Neighbor Lookup）

kd树(K-dimension tree)是⼀种对k维空间中的实例点进⾏存储以便对其进⾏快速检索的树形数据结构。它是一种二叉树，表示对K维空间的划分。构造kd树相当于不断地⽤垂直于坐标轴的超平⾯将K维空间切分，构成⼀系列的K维超矩形区域。KD树每个节点对应一个k维超矩形区域。利用KD树可以减少对大部分数据点的搜索，减少计算量。

可以类比二分查找，把二分查找中的数据点换成K维数据点，这样的划分就变成了⽤超平⾯对k维空间的划分。空间划分就是对数据点进⾏分类，“挨得近”的数据点就在⼀个空间⾥⾯。

5.3 构造方法

  1. 构造根结点，使根结点对应于K维空间中包含所有实例点的超矩形区域
  2. 将分开的区域通过递归，不断找子节点，不断地划分区域。在画分割线时，需要注意要垂直于选定的坐标轴。
  3. 重复上述过程，直到没有尾分化的实例为止。
  • 通常，在选择点时，通常选定某一维的中位数作为切分点，这样得到的KD树是平衡的。
  • 一般选维数，是从低维选到高维。又或者随机选择。更好的方法是在数据比较分散的那一维进行选择（通过方差来衡量）。

5.5 最近领域的搜索

在找对应点时，需要一个队列，来对KD树进行遍历，队列中存储从根结点到最近点路径上的点。判断其他子节点空间中是否有更近的数据点，做法是以查询点为圆心，以最近点的距离为半径画圆，这个圆成为候选超球（candidate hypersphere），如果圆与回溯点的轴相交，则需要将轴的另一边的节点都放入回溯队列中。

总结如下：

  • 从根节点开始，递归地向下搜索，选择距离目标点最近的子节点。
  • 一旦到达叶节点，记录当前最佳点，然后回溯检查可能存在更近点的子节点区域。

下面举两个例子：

查找**(2.1,3.1)**:

在（7,2）点测试，到达（5,4）,再到达（2,3），队列中为[(7,2), (5,4), (2,3)]。从都列中选择(2,3)为当前最佳nearest点，Distance为0.141；

以(2.1,3.1)为圆心，Distance为半径画圆，并不和y=4相交则不必跳到节点(5,4)的右子空间中搜索。故跳过回溯至(5,4)

于是再回溯(7,2)，亦不用跳至(7,2)右子空间中搜索。

至此，队列为空，结束搜索，则最近的邻点为(2,3)

查找**(2,4.5)**:

在（7,2）点测试，到达（5,4）,再到达（4,7），队列中为[(7,2), (5,4), (4,7)]。从都列中选择(4,7)为当前最佳nearest点，Distance为3.202；

以(4,7)为圆心，Distance为半径画圆，与y=4相交则需要考虑左子空间，需要将(2,3)加入队列。然后回溯至(5,4)

计算(5,4)与目标点的距离为3.04，<3.202，所以将(5,4)赋予nearest，并Distance=3.04（距离差不多就用同一张图吧）

于是再回溯(2,3)，为叶子结点，直接判断距离是否更近，计算为1.5，故更新nearest为(2,3)，Distance=1.5

回溯至(7,2)同理画个r = 1.5的圆，不与x=7相交所以不用跳到节点(7.2)右子空间中搜索。

至此队列为空，结束搜索，最近距离为1.5

5.6 总结

kd树的构建过程

  1. 构造根节点
  2. 通过递归的⽅法，不断地对k维空间进⾏切分，⽣成⼦节点
  3. 重复第⼆步骤，直到⼦区域中没有示例时终⽌ 需要关注细节：
    1. 选择向量的哪⼀维进⾏划分；
    2. 如何划分数据

kd树的搜索过程

  1. ⼆叉树搜索⽐较待查询节点和分裂节点的分裂维的值，（⼩于等于就进⼊左⼦树分⽀，⼤于就进⼊右⼦树分 ⽀直到叶⼦结点）
  2. 顺着“搜索路径”找到最近邻的近似点
  3. 回溯搜索路径，并判断搜索路径上的结点的其他⼦结点空间中是否可能有距离查询点更近的数据点，如果有 可能，则需要跳到其他⼦结点空间中去搜索
  4. 重复这个过程直到搜索路径为空

7、KNN算法总结

优点：

  • 简单有效
  • 重新训练的代价低
  • 适合类域交叉样本
  • 适合⼤样本⾃动分类

缺点：

  • 惰性学习
  • 类别评分不是规格化（不是通过概率分类）
  • 输出可解释性不强
  • 对不均衡的样本不擅长
    • 当样本不平衡时，如⼀个类的样本容量很大，⽽其他类样本容量很小时，有可能导致当输⼊⼀个新样本时，该样本的K个邻居中⼤容量类的样本占多数。可以采⽤权值的⽅法（和该样本距离⼩的邻居权值⼤）来改进
  • 计算量较大

8.2 网格搜索

通常情况下，有很多参数是需要⼿动指定的（如k-近邻算法中的K值），这种叫超参数。但是⼿动过程繁杂，所以需要对模型预设⼏种超参数组合。每组超参数都采⽤交叉验证来进⾏评估。最后选出最优参数组合建⽴模型。

API：sklearn.model_selection.GridSearchCV(estimator,param_grid=None,cv=None)

  • 对估计器的指定参数值进⾏详尽搜索
  • estimator：估计器对象
  • param_grid：估计器参数(dict){“n_neighbors”:[1,3,5] }
  • cv：指定⼏折交叉验证
  • fit：输⼊训练数据
  • score：准确率
  • 结果分析：
    • bestscore__：在交叉验证中验证的最好结果
    • bestestimator：最好的参数模型
    • cvresults：每次交叉验证后的验证集准确率结果和训练集准确率结果
